# RAG Chatbot API

A document question-answering microservice built with **FastAPI**, **local embeddings**, **ChromaDB**, and **Claude**. Upload your documents (PDF / TXT / MD), then ask questions and get answers grounded in your content вЂ” **with cited sources**.

This is a Retrieval-Augmented Generation (RAG) system: instead of letting the LLM answer from memory (and hallucinate), it retrieves the most relevant chunks from your documents and instructs the model to answer **only** from that context.

## How RAG works here

```
INGESTION (once per document)
  file в”Ђв”Ђв–¶ extract text в”Ђв”Ђв–¶ chunk в”Ђв”Ђв–¶ embed (vector) в”Ђв”Ђв–¶ store in ChromaDB

QUERY (every question)
  question в”Ђв”Ђв–¶ embed в”Ђв”Ђв–¶ find nearest chunks в”Ђв”Ђв–¶ feed to Claude as context в”Ђв”Ђв–¶ cited answer
```

The core idea: tell the LLM *"don't make things up вЂ” answer based on these retrieved passages."* This reduces hallucination and makes answers auditable.

## Tech stack

| Layer        | Choice                                   | Why                                                |
|--------------|------------------------------------------|----------------------------------------------------|
| API          | FastAPI + Uvicorn                        | Modern, fast, automatic OpenAPI docs               |
| Embeddings   | `intfloat/multilingual-e5-small` (local) | Free, runs offline, supports Turkish + English     |
| Vector DB    | ChromaDB                                 | Local, persistent, stores metadata for citations   |
| LLM          | Claude (`claude-opus-4-8`)               | Answer generation                                  |
| PDF parsing  | pypdf                                     | Text extraction                                    |

## Setup

```bash
# 1. (Recommended) create a virtual environment
python3 -m venv .venv && source .venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure your API key
cp .env.example .env
#   then edit .env and set ANTHROPIC_API_KEY=sk-ant-...
#   (get one at https://console.anthropic.com)

# 4. Run
./run.sh            # or: uvicorn app.main:app --reload --port 8000
```

Open **http://localhost:8000/docs** for the interactive API.

---

## API usage

### Upload a document
```bash
curl -X POST http://localhost:8000/ingest \
  -F "file=@examples/sirket_politikalari.txt"
```

### Ask a question
```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"question": "How many vacation days do employees get?"}'
```

Response:
```json
{
  "answer": "Full-time employees get 20 paid vacation days per year [1]...",
  "sources": [
    {"index": 1, "source": "sirket_politikalari.txt", "score": 0.85, "preview": "..."}
  ]
}
```

### Other endpoints
| Method | Path         | Description                  |
|--------|--------------|------------------------------|
| GET    | `/health`    | Health check + chunk count   |
| GET    | `/documents` | List ingested documents      |
| DELETE | `/documents` | Clear all documents          |

## Possible improvements (roadmap)

- **Streaming responses** for the `/chat` endpoint (token-by-token)
- **Re-ranking** retrieved chunks with a cross-encoder for better precision
- **Evaluation harness** to measure answer quality (faithfulness, relevance)
- **Hybrid search** (keyword + semantic)
- **Authentication** and per-user document isolation
- Swap ChromaDB for **Qdrant / pgvector** for production scale
```
